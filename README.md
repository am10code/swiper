# Swiper — расширение для задач и фокуса

Расширение для Chrome/Chromium с управлением задачами, карточкой фокуса и режимом Свайпер.

## Возможности

- Создание, редактирование и завершение задач, быстрые действия и контекстное меню.
- Категории и срочность (обычный/высокий), дедлайны с подсветкой.
- Отдельная приоритизация порядка задач через попарные сравнения (экран «Что важнее»): числовой ранг не показывается в UI, но используется для сортировки задач во всех основных секциях.
- Разделы задач: просроченные, на сегодня, позже, «Без даты» (по умолчанию свёрнут); при создании задачи с пустым полем дедлайна подставляется «Сегодня». Режим отображения «все» или «только сегодня».
- В хедере есть индикатор (иконка волшебной палочки) для неприоритизированных задач: если есть хотя бы одна активная задача без ранга, индикатор виден; после завершения последнего сравнения скрывается.
- Режим ввода новых задач в настройках: «Поле внизу» (по умолчанию) или «FAB»; при FAB нижнее поле открывается по клику на кнопку `+` справа внизу.
- Поиск по задачам и фильтрация по категории/приоритету.
- Полная карточка задачи: следующие шаги (drag & drop), лог, фокус‑таймер, поле ссылки с валидацией `http/https`; в меню «Больше опций» можно **сбросить вес задачи** (обнулить скрытый ранг приоритизации `priorityRank` — задача снова попадает в пул неприоритизированных, срочность high/medium не меняется).
- Редактирование задачи выполняется через полную карточку; в списках задач кнопка карандаша скрыта.
- В блоке «Следующий шаг»: клик по строке шага переключает выполненность, корзина удаляет шаг.
- У повторяющихся задач рядом с названием показывается серый символ повтора `↻` (индикатор, без действия).
- Режим «регулярное участие»: «Выполнено на сегодня» с переносом дедлайна на период регулярности.
- В редактировании карточки: быстрые дедлайны «Сегодня», «Завтра», «На следующей неделе» (следующий понедельник); поле периода регулярности видно только при включенном «Регулярное участие».
- **Автодедлайн «Сегодня» в полной карточке** (только незавершённые задачи): перенос на сегодняшнюю локальную дату выполняется **только если текущий дедлайн просрочен** (строго раньше сегодня). При дедлайне «сегодня», в будущем или без даты дедлайн из‑за активности в карточке не меняется. Триггеры те же: следующие шаги (добавление, отметка, удаление, порядок), лог (включая «Выполнен шаг…»), **Старт** фокус‑помодоро, смена названия; в форме «Редактировать» — при изменении приоритета, ссылки или регулярности дата из поля перезаписывается на «сегодня» **только для просроченной** задачи; если меняли **только** дедлайн — сохраняется выбранная дата. Явные действия вроде «Перенести на следующую неделю» по-прежнему задают дедлайн вручную.
- Кнопка «Перейти к следующей задаче» в полной карточке на странице задач проходит очередь: сначала просроченные (по приоритету), затем сегодня (по приоритету), затем позже (по дедлайну; при равном дедлайне — по приоритету), затем без дедлайна.
- Попап подтверждения удаления задачи в полной карточке (`#taskDeleteModal`) всегда отображается поверх карточки, чтобы кнопку «Удалить» было можно нажать и не приходилось закрывать режим.
- Режим ФЛОУ: кнопка «ФЛОУ» в хедере (`main.html`) переключает на полноэкранный экран с одной задачей в виде полной карточки; порядок задач — объединенный список «просрочено + сегодня» по приоритету, затем задачи с дедлайном завтра и позже по дедлайну (при равном дедлайне — по приоритету), затем задачи без дедлайна. При входе в ФЛОУ сохраняется **снимок** этого порядка (id задач): внутри одного прохода «Перейти к следующей задаче», «Выполнено на сегодня» и «Отметить выполненной» ведут к следующей задаче **по снимку** (смена дедлайна в карточке не перескакивает очередь по свежей сортировке). После задачи с **последним** id в снимке выполняется **переснимок** по актуальным данным и открывается первая задача нового порядка — как при повторном входе в ФЛОУ, без закрытия карточки; дальше снова обход по новому снимку. Повторный клик по «ФЛОУ», когда режим уже открыт, не перезапускает вход. Выход только через бургер-меню (выбор любого раздела сбрасывает снимок и закрывает карточку). Подтвержденное удаление задачи в полной карточке в режиме ФЛОУ также использует обход по снимку: после удаления открывается следующая задача по снимку, а с последнего id выполняется переснимок; если задач больше нет, карточка закрывается и ФЛОУ завершается.
- Помодоро с историей сессий, итоговым временем и фоновым звуком (код использует пути `assets/audio/pomodoro-ambience.mp3` и `assets/audio/bell.wav`).
- В настройках помодоро используется структурированная форма с подсказками по каждому полю, а после сохранения показывается короткий статус; некорректные числовые значения автоматически приводятся к допустимому диапазону.
- Свайпер: разбор очереди задач свайпами, undo и горячие клавиши (`←`, `→`, `Z`/`Backspace`, `Delete`).
- Раздел «Часто откладываемые» на основе счетчика переносов.
- Импорт/экспорт задач в `.swiper`/`.json` с обратной совместимостью (обязателен только `text`, лишние поля не ломают импорт).
- Уведомления по дедлайнам и по завершению помодоро.
- Глобальная настройка `Логировать выполнение шагов`: при включении в лог задачи добавляется запись `Выполнен шаг ...` с временем закрытия шага; состояние чекбокса сохраняется сразу при переключении.
- Настройка режима добавления задач сохраняется сразу при переключении radio-кнопки.
- Выполненные задачи группируются по датам завершения («Сегодня», «Вчера», дата).

## Установка

1. Откройте Chrome и перейдите в `chrome://extensions/`.
2. Включите «Режим разработчика» (Developer mode).
3. Нажмите «Загрузить распакованное расширение» (Load unpacked).
4. Выберите папку с расширением.

## Сборка CRX (фиксированная команда)

Публичная документация не содержит путь к ключу и приватную команду сборки.

## Точки входа

- По клику на иконку расширения service worker открывает `main.html` в новой вкладке (`chrome.action.onClicked` в `background/background.js`).
- `manifest.json` не содержит `action.default_popup`.
- `main.html` — production entrypoint основного интерфейса.
- `swiper.html` — secondary standalone-страница режима Свайпер.
- `popup/popup.html` — legacy-страница (не production entrypoint и не action popup).

## Архитектура

- `popup/storage.js` — работа с данными в `chrome.storage.local`; при обновлении лога и следующих шагов через API карточки дедлайн **просроченной** активной задачи может выставляться на сегодня (`updateTaskCardData` с `syncDeadlineToToday`).
- `popup/popup.js` — логика разделов и списков задач + page-specific bootstrap для `main.html`/`popup/popup.html`; снимок очереди ФЛОУ (`flowSessionOrderedIds`, пересъёмка при новом цикле), экспорт `getCurrentSectionName`, `getFlowSessionOrderedIds`, `refreshFlowSessionSnapshot`.
- `popup/task-card.js` — полная карточка задачи и помодоро (идемпотентная инициализация через `initTaskCardPage`); навигация «следующая» в ФЛОУ по снимку сессии (`resolveNextFlowTaskId`).
- `popup/modules/dialog-service.js` — модальные диалоги (showAlert/showConfirm) вместо alert/confirm.
- `popup/modules/import-export.js` — импорт/экспорт задач.
- `popup/modules/prioritization.js` — движок попарной приоритизации с инкрементальной вставкой (минимальное количество сравнений для новых задач).
- `popup/modules/swiper/swiper.js` — ядро Свайпера.
- `popup/modules/swiper/swiper-init.js` — bootstrap `initSwiperPage(...)` для standalone и встроенного режима.
- `popup/modules/swiper/swiper-nav.js` — standalone-навигация только для `swiper.html`.
- `background/background.js` — сервис‑воркер уведомлений и алармов (в т.ч. fallback-иконка уведомлений через data URL); при изменении задач напоминания по дедлайнам синхронизируются инкрементально с debounce 400 ms.

## Текущее состояние ассетов

- Для полной работы расширения нужны: **иконки** `assets/icons/icon16.png`, `icon48.png`, `icon128.png` (манифест, уведомления) и при необходимости **аудио** `assets/audio/bell.wav`, `assets/audio/pomodoro-ambience.mp3` (помодоро и уведомления). В репозитории по умолчанию только SVG в `assets/icons/` и инструкции в `assets/icons/README.txt` и `assets/audio/README.txt`.
- PNG можно получить конвертацией из SVG (см. `assets/icons/README.txt`). Ожидаемые аудио-файлы перечислены в `assets/audio/README.txt`.

## Отладочные логи

- В коде используется общий логгер `shared/logger.js`; уровень `debug` по умолчанию не выводится. Чтобы включить отладочный вывод: в консоли страницы расширения выполнить `localStorage.setItem('swiper_debug', '1')` или задать `window.__SWIPER_DEBUG__ = true` до загрузки скриптов.
